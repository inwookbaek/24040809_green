JWT(JSON Web Token)

JWT는 화면이 필요 없기 때문에 기존소스를 참고해서 새로운 프로젝트를 생성

화면 없이 Ajax와 JSON을 이용해서 데이터를 주고 받는 구조에서 HttpSession이나 쿠키를 이용하는 기존의 인증방식에 제한받게 된다. 이를 해결하기 위해 인증받은 사용자들은 특정한 문자열(토큰)을 이용하는데 가장 많이 사용하는 것이 JWT(JSON Web Token)이다.

API서버

  • API서버는 쉽게 '필요한 데이터만 제공하는 서버'를 의미
  • API서버는 화면을 제공하는 것이 아니라 필요한 데이터를 호출하고 결과를 반환하는 방식으로 동작
  • 즉, 가장 큰 특징은 '화면을 제공하지 않는다!!'
  • 이와는 다르게 SSR(Server Side Rendering)은 화면을 서버에서 만들어 전송하는 방식. JSP와 Thymeleaf가 이에 해당한다.
  • API서버는 화면구성을 클라이언트에서 처리하고 서버는 순수한 데이터만 전송
  • 이러한 구성을 Client Side Redering(CSR)이라 한다.
  • 또한, 무상태(stateless)라는 특징도 있다.

토큰기반의 인증

Access Token / Refresh Token의 의미
  • Token을 API서버에서는 Access Token이라고 한다.
  • Token은 특정한 자원에 접근할 권한이 있는지를 검사하기 위한 용도
  • Access Token을 이용할 때는 최대한 유효시간을 짧게 지정하고 사용자에게는 새로 발급받을 수 있는 Refresh Token을 생성한다.
